Инструментарий

Jenkins позволяет работать над непрерывным развертыванием или непрерывным автоматическим тестированием ПО.

Jenkins – это инструмент непрерывной интеграции, который чаще всего используется для разработки программного обеспечения. Это среда автоматизации, которая выполняет повторяющиеся задания. Jenkins может выполнять и контролировать выполнение команд на удаленных системах, а также всего того, что можно выполнить из командной строки. С помощью вспомогательных плагинов Jenkins объединяет электронную почту, TestNG и другие инструменты.

Allure report замечательный инструмент для отчетности прохождения автотестов.

Nexus – является очень популярным менеджером репозиториев (repository manager) от компании Sonatype. Он позволяет поднимать маленький Maven Central внутри вашей компании.

Docker – это программная платформа для быстрой разработки, тестирования и развертывания приложений. Docker упаковывает ПО в стандартизованные блоки, которые называются контейнерами. Каждый контейнер включает все необходимое для работы приложения: библиотеки, системные инструменты, код и среду исполнения. Благодаря Docker можно быстро развертывать и масштабировать приложения в любой среде и сохранять уверенность в том, что код будет работать.

WebLogic это замечательный сервер приложений. Который, зачастую, очень часто встречается в веб приложениях.

Тестирование UI (веб-интерфейса)

Для автоматизации UI (веб-интефейса) применяется язык Java.

Для автоматизации UI применяется самописный фрэймворк, в котором используются другие фрэймворки, такие как Spring Framework, MyBatis, Serenity, JUnit, Selenide

Применяются паттерны Page Object, Page Factory и др..

Тестирование web-сервисов (API)

Автоматизация web-сервисов осуществляется очень просто, с использованием интересной программки SoapUI и библиотеки JDBC, которая позволяет подключиться к нужной Базе Данных (например: Oracle и MS SQLServer).

Другой подход тестирования API это применение языка Java и библиотеки TestNG
